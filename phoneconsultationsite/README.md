# 휴대폰싸게파는사람들 — 셀프 견적 상담 사이트

고객이 **통신사 → 기종 → 가입유형 → 요금제 → 지원방식 → 부가서비스**를 직접 골라가며
월 예상 납부액을 확인하는 셀프 견적 사이트입니다. 관리자는 매일 엑셀 파일 하나만
업로드하면 오늘의 판매 기종·요금제·정책이 모든 고객 화면에 즉시 반영됩니다.

- **고객 페이지**: `index.html`
- **관리자 페이지**: `admin.html` (비밀번호 보호, 검색엔진 노출 안 됨)

---

## 1. 먼저 이것부터 하세요 — Firebase 설정하기

엑셀을 올리는 즉시 모든 손님 화면에 반영되게 하려면 무료 데이터베이스(Firebase)를
연결해야 합니다. **아직 연결하지 않아도 사이트 자체는 정상 동작**합니다 — 다만
관리자 페이지에서 저장한 내용이 그 브라우저에만 남고, 다른 손님에게는 보이지 않습니다.
(관리자 페이지 상단에 "Firebase 미연결" 안내가 뜨면 아직 이 단계가 안 된 것입니다.)

### 1) 프로젝트 만들기
1. https://console.firebase.google.com 접속 → 구글 계정으로 로그인
2. **프로젝트 추가** 클릭 → 이름 입력(예: `gonggu-phone`) → 애널리틱스는 꺼도 됩니다 → **프로젝트 만들기**

### 2) 웹 앱 등록하고 설정값 받기
1. 프로젝트 화면에서 **`</>`(웹)** 아이콘 클릭
2. 앱 닉네임 아무거나 입력 → **앱 등록**
3. 화면에 나오는 `firebaseConfig = { apiKey: "...", ... }` 값을 그대로 복사
4. 이 저장소의 `js/firebase-init.js` 파일을 열어 맨 위 `firebaseConfig` 객체를
   복사한 값으로 통째로 교체 → 저장

### 3) Firestore(데이터베이스) 켜기
1. 왼쪽 메뉴 **빌드 > Firestore Database** → **데이터베이스 만들기**
2. 위치는 아무거나(가까운 지역 권장, 예: `asia-northeast3`) → **테스트 모드로 시작** 선택
3. 만들어진 후 **규칙(Rules)** 탭으로 이동해서 아래 내용으로 전체 교체 후 **게시**:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /config/site {
         allow read: if true;
         allow write: if request.auth != null;
       }
     }
   }
   ```
   (손님은 누구나 읽을 수 있고, 로그인된 사람만 쓸 수 있다는 규칙입니다.)

### 4) 익명 로그인 켜기
1. 왼쪽 메뉴 **빌드 > Authentication** → **시작하기**
2. **Sign-in method** 탭 → **익명(Anonymous)** 선택 → 사용 설정 → 저장

여기까지 하면 준비 끝입니다. `admin.html`에서 엑셀을 올리고 "적용"을 누르면
몇 초 안에 `index.html`을 보고 있는 모든 손님 화면에 자동으로 반영됩니다.

> Firebase 무료(Spark) 요금제로 충분합니다. 이 정도 트래픽으로는 비용이 발생하지 않습니다.

## 2. 매일 정책 반영하기 (엑셀)

1. `admin.html` 접속 → 비밀번호 입력 (기본값 `gonggu2026`, **꼭 바꾸세요** — 아래 4번 참고)
2. **① 엑셀 서식 다운로드**로 서식 파일을 받습니다 (`excel-template/휴대폰정책_서식.xlsx`)
3. 엑셀에서 시트 4개를 채웁니다 (시트 이름·열 제목은 바꾸지 마세요):

| 시트 | 내용 |
|---|---|
| **정책** | 오늘 번호이동/기기변경/공시/선약 허용 여부, 매장 추가지원금, 선택약정 할인율, 할부 이자율, 할부개월 |
| **기종** | 기종명, 용량, 출고가, 카테고리, 오늘판매(Y/N), 통신사별 공시지원금 |
| **요금제** | 통신사별 요금제명, 월정액, 대표요금제 여부 |
| **부가서비스** | 통신사별 부가서비스명, 유지개월, 가입 시 매장이 추가로 지원하는 금액 |

- **기종** 시트에서 `SKT공시`/`KT공시`/`LGU+공시` 칸을 비워두면 그 통신사에서는
  판매하지 않는 기종으로 처리되어 고객 화면에서 자동으로 빠집니다. (예: SKT 전용 모델)
- **오늘판매(Y/N)**를 `N`으로 바꾸면 그 기종은 오늘 고객 화면에서 숨겨집니다.
- **부가서비스**는 고객이 선택하면 매장지원금만큼 기기값이 추가로 할인됩니다.

4. 완성한 파일을 `admin.html`의 업로드 칸에 끌어다 놓으면 내용을 미리보기로
   보여줍니다. 확인 후 **"이 내용으로 사이트에 적용하기"**를 누르면 끝입니다.

## 3. 매장 정보 바꾸기

`js/store-data.js` 맨 위 `STORE` 객체를 수정하세요.

```js
const STORE = {
  siteName: "휴대폰싸게파는사람들",
  bizName: "공구폰(휴대폰싸게하는사람들)",
  phone: "010-4191-1918",
  phoneDigits: "01041911918",   // 전화 걸기 버튼용, 하이픈 없이
  address: "광주광역시 광산구 장신로 134 1층 106호",
  kakaoUrl: "http://pf.kakao.com/_xkvzXn/chat",
  prepaidUrl: "https://bbangbbangtel.github.io/Prepaid-mobile-landing",
  hours: "..."
};
```

`models`/`plans`/`addons`의 임시값(데모 데이터)도 같은 파일 `DEMO_SITE_DATA`에 있는데,
이건 **Firebase 연결 전이거나 아직 엑셀을 한 번도 안 올렸을 때만** 보여지는 값입니다.
엑셀을 한 번이라도 적용하면 그 이후로는 항상 엑셀로 올린 내용이 우선 표시됩니다.

## 4. 관리자 비밀번호 바꾸기

기본 비밀번호는 `gonggu2026`입니다. **반드시 바꿔서 사용하세요.**

1. 아무 페이지에서 브라우저 개발자 도구(F12) → Console 탭 열기
2. 아래 코드에서 `새비밀번호` 부분을 원하는 비밀번호로 바꿔 실행:
   ```js
   const pw = "새비밀번호";
   const buf = await crypto.subtle.digest("SHA-256", new TextEncoder().encode(pw));
   console.log([...new Uint8Array(buf)].map(b=>b.toString(16).padStart(2,"0")).join(""));
   ```
3. 출력된 긴 문자열을 복사
4. `js/admin.js` 파일에서 `ADMIN_PASSWORD_HASH` 값을 이 문자열로 교체

> 참고: 이 비밀번호는 손님/직원의 실수 접근을 막는 용도이지, 완벽한 보안장치는
> 아닙니다. `admin.html` 주소 자체를 아무에게나 공유하지 않는 것이 중요합니다.

## 5. 로컬에서 미리보기

이 사이트는 `<script type="module">`을 사용하므로 파일을 더블클릭해서 여는 것(file://)은
동작하지 않습니다. 아래처럼 로컬 서버로 열어야 합니다.

```bash
python3 -m http.server 8000
# 이후 브라우저에서 http://localhost:8000 접속
```

## 6. GitHub Pages로 배포하기

1. 이 저장소를 GitHub에 올린 뒤 **Settings → Pages**
2. Source: `Deploy from a branch` → 브랜치 선택 → 저장
3. 몇 분 뒤 `https://<계정명>.github.io/<저장소명>/` 에서 고객 페이지,
   `.../admin.html` 에서 관리자 페이지 접속 가능

## 주의

- 사이트에 표시되는 금액은 관리자가 입력한 값 기준의 **예상 견적**이며,
  실제 조건은 상담 시 확정됩니다.
- Firebase를 연결하기 전까지는 데모 데이터가 보이며, 관리자 저장 내용도
  그 브라우저에만 남습니다. 실제 운영 전 꼭 1번 단계를 완료하세요.
- 관리자 비밀번호는 반드시 기본값에서 변경하세요 (4번 참고).
